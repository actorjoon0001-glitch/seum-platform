# CLAUDE.md

세움 플랫폼(통합 업무 포털) 저장소에서 Claude가 따라야 할 규칙.

## Git 워크플로 (필수)

이 저장소의 라이브 사이트는 **Netlify가 `main` 브랜치를 자동 배포**한다.
따라서 모든 변경은 아래 흐름을 **반드시** 따른다.

1. **`main`에 직접 push 하지 않는다.**
2. 변경마다 `origin/main` 기준으로 **새 브랜치**를 만든다.
3. 변경 후 **`main`으로 PR**을 연다.
4. **빌드/타입체크가 통과할 때만** 머지한다:
   - `npm run build`
   - `npx tsc --noEmit`
5. 통과하면 PR을 **squash로 자동 머지**한다 → Netlify가 `main`을 자동 배포.
6. 빌드/타입체크가 **실패하면 머지하지 말고** 먼저 고친다.

> 요약: **새 브랜치 → PR → (빌드 통과 시) 자동 머지 → 자동 배포.**
> 검토 없이 깨진 코드가 배포되지 않도록, 머지 전 빌드 통과는 생략하지 않는다.

## 프로젝트 메모

- 루트(`/`)는 `/portal`로 리다이렉트되는 **내부 업무 포털**이다(고객용 소개 사이트 아님).
- "자주 사용하는 시스템" 런처 카드는 `features/portal/config/systems.ts`의 `SYSTEMS` 배열에서 관리한다.
  - 외부 서비스가 실제로 동작하면 `ready: true` + 실제 `serviceUrl`로 둔다(클릭 시 임베드).
  - 아직 준비 안 된 서비스는 `ready: false`로 두면 "연결 준비중" 안내가 뜬다.
