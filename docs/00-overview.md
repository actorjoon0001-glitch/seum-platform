# 00. 제품 개요 (Overview)

## 1. 비전

> "건축 업체의 영업·설계·시공 전 과정을 디지털 한 줄로 잇는다."

건축/이동식주택/전원주택/인테리어 업체는 대부분
**엑셀 견적 + 카톡 상담 + 종이 계약**으로 운영된다.
BuildLab는 이 과정을 하나의 SaaS로 통합해
**상담 → 견적 → 3D 설계 → 계약 → 시공**의 리드타임을 줄이고
전환율과 고객 경험을 높인다.

## 2. 타깃 사용자 (Persona)

| 페르소나 | 역할 | 주요 니즈 |
|----------|------|-----------|
| 대표/마스터 | 회사 오너 | 매출/계약 현황을 한눈에, 직원 권한 관리 |
| 영업담당 | 상담/견적 | 빠른 견적, 고객 파이프라인 관리 |
| 설계담당 | 도면/3D | 모델 기반 3D 설계, 옵션 반영 |
| 시공담당 | 현장 | 시공 단계/사진/일정 업데이트 |
| 정산담당 | 회계 | 계약금/중도금/잔금 추적 |
| (간접) 고객 | 의뢰인 | 견적서/계약서 수신, 진행 상황 확인 |

## 3. 범위 (Scope)

### In Scope (v1 ~ v3)
- 멀티테넌트 SaaS (업체별 데이터 격리)
- CRM, 카탈로그, 실시간 견적, 견적서 PDF
- 3D 설계 뷰어/에디터 (가구·창·문 배치)
- 계약/시공 관리, 파일 업로드
- RBAC, 대시보드

### Out of Scope (별도 로드맵)
- 결제(PG) 연동, 실제 전자서명 인증 (구조만 확장 가능하게)
- AI 자동설계 (Scene Spec 구조로 미리 대비)
- 모바일 네이티브 앱 (반응형 웹 우선)
- 회계/세무 시스템 연동

## 4. 용어 사전 (Glossary)

| 용어 | 정의 |
|------|------|
| **Organization (Org / Tenant)** | 플랫폼을 사용하는 업체 단위. 모든 데이터 격리의 기준. |
| **Member** | Org에 소속된 사용자(직원). 1명이 여러 Org 소속 가능. |
| **Lead / Customer** | 상담 유입 고객. 파이프라인 상태를 가짐. |
| **Model** | 평형별 건축 상품(10평/14평/맞춤설계 등). |
| **Option** | 모델에 추가하는 항목(포치/데크/다락/태양광 등). |
| **Quote** | 모델 + 옵션 조합으로 산출된 견적. |
| **Scene Spec** | 3D 설계를 표현하는 직렬화 JSON. AI 설계 확장의 입력/출력 포맷. |
| **Contract** | 계약(계약금/중도금/잔금 구성). |
| **Project** | 계약 이후 진행되는 시공 건. 시공 단계를 가짐. |

## 5. 상태 머신 요약

### 고객(Customer) 파이프라인
```
신규 → 상담중 → 견적발송 → 계약대기 → 계약완료 → 시공중 → 완료
```

### 시공(Project) 단계
```
계약 → 설계 → 허가 → 생산 → 운송 → 설치 → 완료
```

> 두 상태 머신은 **계약완료/계약** 지점에서 연결된다.
> 고객이 `계약완료`가 되면 Project가 생성되어 시공 단계가 시작된다.

## 6. 성공 지표 (North Star)

- 업체당 **월 견적 발송 건수**
- 견적 → 계약 **전환율**
- 상담 등록 → 계약완료까지 **평균 리드타임**
